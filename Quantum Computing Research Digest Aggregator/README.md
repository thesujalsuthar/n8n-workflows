# Quantum Computing Research Digest Aggregator

This workflow automatically aggregates, summarizes, categorizes, and emails a daily digest of the latest quantum computing research papers from arXiv and IEEE Xplore APIs.

---

## Workflow Overview

- **Trigger:** Runs daily at 8:00 AM (Cron configured)
- **Sources:** 
  - arXiv API (quantum computing papers)
  - IEEE Xplore API (quantum computing papers)
- **Processing:**
  - Parses and normalizes data formats from both sources
  - Uses OpenAI to generate concise summaries for each paper abstract
  - Categorizes summaries by quantum computing topics based on keywords
  - Groups papers by category
- **Output:** Formats grouped summaries into an HTML digest email
- **Distribution:** Sends the digest via email to a predefined subscribers list

---

## Node Descriptions

### 1. Daily Trigger
- **Type:** Cron
- **Schedule:** Runs every day at 08:00 AM (server time)
- **Purpose:** Initiates the workflow once daily

### 2. Fetch arXiv Quantum Papers
- **Type:** HTTP Request
- **Endpoint:** [arXiv API](http://export.arxiv.org/api/query)
- **Query:**
  - Search for "quantum computing"
  - Retrieve latest 10 results sorted by submission date
- **Response Format:** XML string  

### 3. Fetch IEEE Quantum Papers
- **Type:** HTTP Request
- **Endpoint:** [IEEE Xplore API](https://ieeexploreapi.ieee.org/api/v1/search/articles)
- **Query Parameters:**
  - `querytext`: quantum computing
  - `max_records`: 10
  - `format`: json
  - `apikey`: Your IEEE API key (stored in credentials)
- **Response Format:** JSON

### 4. Parse arXiv XML
- **Type:** Function
- **Functionality:** Parses XML response from arXiv API using `fast-xml-parser` package and extracts:
  - Title
  - ID
  - Summary (abstract)
  - Authors (list of names)
  - Published date

### 5. Parse IEEE JSON
- **Type:** Function
- **Functionality:** Normalizes IEEE JSON response, extracting for each article:
  - Title
  - ID
  - Summary (abstract)
  - Authors (list of full names)
  - Published date

### 6. Summarize Abstract with AI
- **Type:** OpenAI Node
- **Functionality:** Sends the paper abstract to OpenAI API with prompt to summarize in 2-3 concise sentences emphasizing key findings.
- **Notes:** Uses your OpenAI API key stored in credentials.

### 7. Categorize by Topic
- **Type:** Function
- **Functionality:** Analyzes summary text and categorizes papers into topics by checking the presence of keywords related to:
  - Quantum Algorithms
  - Quantum Hardware
  - Quantum Error Correction
  - Quantum Cryptography
  - Quantum Simulation
  - Quantum Communications
- **If no keywords found, the paper is marked "Uncategorized".**

### 8. Group Summaries by Topic
- **Type:** Function
- **Functionality:** Groups all summarized papers by their assigned topics. Each group contains:
  - Title
  - Summary
  - Authors (joined as a string)
  - Published date

### 9. Format Digest Email
- **Type:** Function
- **Functionality:** Creates a formatted HTML email body containing:
  - Title with the current date
  - Sections per topic with a list of papers, showing:
    - Title (bold)
    - Published date
    - Authors
    - Summary

### 10. Get Subscribers
- **Type:** Function
- **Functionality:** Returns a static list of subscriber emails to receive the digest.
- **Emails included:** `alice@research.org`, `bob@quantummail.com`
- **Can be modified to dynamically fetch subscribers from a data source.**

### 11. Send Digest Email
- **Type:** Email Send
- **Functionality:** Sends the digest email with:
  - From: `research-digest@quantumhub.org`
  - To: List of subscribers or fallback email in credentials
  - Subject: `Daily Quantum Computing Research Digest`
  - Plain text fallback message
  - HTML content generated by "Format Digest Email"
- **Requires SMTP credentials with name `SMTP Email`**

## Credentials Required

- **OpenAI API Key** (for summarizing abstracts)
- **IEEE Xplore API Key** (to access IEEE articles)
- **SMTP Email Credentials** (to send emails)
- Optional fallback recipient email configured in credentials under `EMAIL_TO`

---

## Setup Instructions

1. **Import Workflow:**
   - Import this workflow JSON into your n8n instance.

2. **Configure Credentials:**
   - Add your OpenAI API key to the OpenAI node credentials.
   - Add your IEEE Xplore API key.
   - Configure SMTP credentials for outgoing email.

3. **Modify Subscribers:**
   - Update the "Get Subscribers" node to add/remove recipients or replace with dynamic subscriber retrieval logic.

4. **Schedule:**
   - Verify and adjust the cron expression in the "Daily Trigger" node to your preferred time zone or time.

5. **Run Workflow:**
   - Activate the workflow.
   - It will automatically run daily, retrieving, processing, and sending the digest.

## Customization Opportunities

- Expand the number of fetched papers by modifying `max_results`/`max_records`.
- Add more sources or APIs to collect research papers.
- Enhance AI summarization prompt for style or length.
- Store subscriber emails in a database or external storage for dynamic fetching.
- Customize email templates to your branding or add attachments.
